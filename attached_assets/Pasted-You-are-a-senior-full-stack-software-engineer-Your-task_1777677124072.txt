You are a senior full-stack software engineer.

Your task is to build a complete, production-ready web application called "Study Group Finder" for a university.

The app helps students create, discover, and join study groups.

========================
CORE FEATURES
========================

1. Authentication:
- Students can sign up and log in using email
- Secure authentication (JWT or session-based)
- Each user has a profile:
  - Name
  - Faculty
  - Academic year
  - Subjects of interest

2. Study Groups:
- Users can create study groups
- Each group includes:
  - Subject / Course name
  - Description
  - Group type: Online or Offline
  - If Offline → include location (building/room)
  - Date and time
  - Maximum number of members
- Users can join and leave groups

3. Dashboard:
- List all available groups
- Filtering:
  - By subject
  - By type (online/offline)
  - By date

4. Group Page:
- Show full group details
- List of members
- Join/Leave button

5. Chat System:
- Each group has its own chat
- Real-time or near real-time messaging
- Messages linked to users and groups

6. Admin Panel:
- Admin role system
- Admins can:
  - Delete users
  - Delete groups
  - Moderate messages

========================
TECH STACK REQUIREMENTS
========================

Frontend:
- React or Next.js
- Clean, modern UI (student-friendly)
- Responsive design

Backend:
- Node.js with Express

Database:
- PostgreSQL (STRICT REQUIREMENT — do NOT use MongoDB or Firebase)

========================
DATABASE DESIGN (IMPORTANT)
========================

Design a normalized relational schema with proper constraints.

Tables must include:

- Users
  id (PK)
  name
  email (unique)
  password
  faculty
  academic_year

- Groups
  id (PK)
  title
  subject
  description
  type (online/offline)
  location (nullable)
  date_time
  max_members
  created_by (FK → Users.id)

- GroupMembers (many-to-many)
  id (PK)
  user_id (FK → Users.id)
  group_id (FK → Groups.id)

- Messages
  id (PK)
  group_id (FK → Groups.id)
  user_id (FK → Users.id)
  content
  created_at

- Roles (or Admin system)
  id (PK)
  user_id (FK → Users.id)
  role (admin/student)

Requirements:
- Define primary keys and foreign keys
- Add constraints (NOT NULL, UNIQUE where needed)
- Enforce relationships properly

Also:
- Provide SQL CREATE TABLE statements

========================
API DESIGN
========================

Create RESTful API endpoints such as:

Auth:
- POST /register
- POST /login

Users:
- GET /profile

Groups:
- GET /groups
- POST /groups
- GET /groups/:id
- POST /groups/:id/join
- POST /groups/:id/leave

Messages:
- GET /groups/:id/messages
- POST /groups/:id/messages

Admin:
- DELETE /users/:id
- DELETE /groups/:id
